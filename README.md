# Жемчужина Алтая — монорепозиторий

Интернет-магазин натуральных продуктов Алтая: веб-витрина, админ-панель (Strapi),
бэкенд (NestJS + PostgreSQL), двусторонняя интеграция с Эвотором, онлайн-оплата
с фискализацией («Цифровая касса»), Mini App в мессенджере MAX.

Разработка ведётся по документу **«ПЛАН РАЗРАБОТКИ.md»** (92 шага, 6 блоков),
составленному по финальному ТЗ v1.0 от 05.07.2026.

## Структура репозитория

```
altay-gem/
  apps/
    backend/        # NestJS: каталог, остатки, заказы, промокоды, интеграции (шаг 0.2+)
    strapi/         # Strapi: админ-панель контента (этап 1)
    frontend/       # витрина (TanStack Start) — вшита в монорепо, весь код здесь
  infra/
    docker/         # Dockerfile-ы по сервисам
    nginx/          # конфиги реверс-прокси + TLS
    compose/        # docker-compose.*.yml
    certs/          # ПУБЛИЧНЫЕ корневые CA (Минцифры/Эвотор); приватные ключи запрещены
    backup/         # скрипты бэкапа/восстановления БД
  .github/workflows/  # CI/CD (шаг 0.11)
  .env.example      # шаблон переменных окружения (реальный .env не коммитится)
  Makefile          # единые команды для разнородных приложений
```

## Фронтенд

Витрина (изначально из `Neeklo1606/altai-gem-showcase`, TanStack Start) **вшита прямо
в монорепо** — весь код проекта хранится в одном репозитории `Osmanovvv/altay-gem`.
Отдельного сабмодуля и связи с исходным репозиторием нет.

Локально уже применён фикс: фото товаров переведены с внешнего CDN (Unsplash) на
локальные файлы `apps/frontend/public/img/stock` — надёжнее и подходит под РФ-хостинг.
На Этапе 1 фото и данные переедут на наш API/Strapi.

```bash
git clone https://github.com/Osmanovvv/altay-gem.git
cd altay-gem/apps/frontend && bun install && bun run dev
```

## Управление рабочими областями

Приложения разнородные (NestJS / Strapi / TanStack Start), поэтому workspaces
не используем: **у каждого приложения свой `package.json`**, общие команды — через
корневой `Makefile` (`make help`).

## Требования к окружению

Фиксируются на шаге 0.1a (`.nvmrc`, версия Bun, digest-пины docker-образов).
Ориентир: Node.js 20 LTS, Bun, Docker Engine + `docker compose` v2, Git.

## Секреты

Все секреты (токены Эвотора, платёжного сервиса, Telegram, пароли БД) — **только
в переменных окружения** (`.env`, не коммитится; шаблон — `.env.example`).
Ни одного токена в коде или git — проверяется на ревью и в CI.
